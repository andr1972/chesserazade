# 0.4 — PGN, history, minimal interactive play

Tagged as **v0.4.0**.

The first version of chesserazade that a human can actually *use*
end-to-end: load a PGN, step through it interactively, enter moves
in SAN or UCI, undo, save back to PGN. No engine is attached —
that is 0.5.

## What was added

### `Game` (`include/chesserazade/game.hpp`)
A thin wrapper around `Board8x8Mailbox` that also remembers the
starting position and the sequence of moves played. `play_move`
delegates to `Board::make_move` and appends to history;
`undo_move` pops and calls `unmake_move`; `reset_to_start`
restores the starting position, discarding history.

Composition, not inheritance. `Game` is the foundation the PGN
writer and the `repl` command are built on; the search layer
(0.5+) will use it too for "played PV" display.

### SAN I/O (`include/chesserazade/san.hpp`, `src/io/san.cpp`)
Standard Algebraic Notation — the human move form used in printed
chess books and in PGN. Two entry points:

- `parse_san(board, text)` — resolves SAN against the board's
  legal moves. Handles piece letters, optional disambiguators
  (file / rank / both), capture marker, target square,
  promotion `=X`, and castling (`O-O`, `O-O-O`). Lenient on input:
  strips trailing `+`, `#`, `!`, `?` annotations; accepts
  digit-castle aliases `0-0`, `0-0-0`.
- `to_san(board_before, move)` — emits minimal disambiguation
  (file, else rank, else full square), pawn captures with
  from-file (`exd5`), promotion as `=Q`, and a check / mate
  suffix computed by playing the move and inspecting the
  opponent's reply set. Strict output — no NAGs, castling as
  letters.

### PGN I/O (`include/chesserazade/pgn.hpp`, `src/io/pgn.cpp`)
`PgnGame` = `{ tags, starting_fen?, moves, termination }`. Two
entry points:

- `parse_pgn(text)` — lenient parser: skips `{…}` / `;…` comments,
  `(…)` variations, NAGs (`$1`, `!`, `?`, `!?`, …); tolerates
  move numbers with any dot-count; honors `[SetUp "1"]` +
  `[FEN "…"]` for non-standard starts. SAN moves are resolved
  against a live replay board so illegal moves fail the parse
  with a located error ("move N: …").
- `write_pgn(game, tags, termination)` — emits the Seven-Tag
  Roster (Event / Site / Date / Round / White / Black / Result)
  in canonical order, with missing tags filled as "?". Inserts
  `[SetUp "1"]` + `[FEN]` automatically when the starting
  position is non-standard. Move text wraps at a soft 80-column
  limit.

### CLI: `repl` (alias `play`)
Interactive session over a `Game`. Commands at the prompt:

```
<uci|san>     e2e4, Nf3, O-O, e8=Q, exd5 — UCI and SAN both
              accepted; UCI is tried first when the shape fits.
move <mv>     Same as above; the `move` keyword is optional.
undo          Pop the last move.
show          Render the board (ASCII).
legal         Sorted UCI list of legal moves.
pgn [path]    Save as PGN; stdout if no path.
load <path>   Load a PGN file, replace current game.
reset         Discard history, return to starting position.
help          Print command help.
quit | exit   Leave the session.
```

Starting position is configurable via `--fen` (just like `show`).

## Acceptance criteria (HANDOFF §9 0.4)

- [x] PGN parser and writer (tag pairs, SAN moves, comments
      tolerated on input, NAGs tolerated, variations skipped —
      per HANDOFF).
- [x] Move history via `Game` sitting above `Board`.
- [x] `repl` / `play` subcommand with UCI + SAN input, undo,
      show, save PGN, load PGN.
- [x] Load a sample PGN, step through it, save it back — the
      move list and tags round-trip semantically (verified by
      `PGN: parse -> play -> write -> parse yields the same moves`
      and smoke-tested at the `repl`).

## What is *not* in 0.4

- **Comments are dropped on output.** The parser tolerates
  `{…}` / `;…` so existing PGN files load cleanly, but the
  writer does not attempt to reproduce them. Comment
  round-tripping is a 0.9 concern (annotator output).
- **Variations are dropped.** HANDOFF §9 0.4 says they are
  optional; we skip them on input and never emit them.
- **No engine.** `repl` plays both colors from the keyboard.
  The search layer in 0.5 will add an opposing brain behind
  an optional `auto` flag.
- **SAN input does not require the `+` / `#` suffix.** It is
  tolerated if supplied, but we do not demand it — the user
  should not have to out-think the board to type a move.

## Known deviations

- The Seven-Tag Roster in the writer is always emitted, even
  when the caller did not supply all of it. Missing tags come
  out as `"?"`. This matches PGN strictness: a file without
  STR tags is non-conformant.
