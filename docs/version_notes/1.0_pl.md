# 1.0 — Pobieranie z internetu + przejście po dokumentacji

Otagowane jako **v1.0.0**.

Kamień milowy „można oddać obcej osobie". Silnik z 0.9 był już
użyteczny; 1.0 dodaje ostatni afordans ze światem zewnętrznym
(pobieranie gier / puzzli z internetu) i porządkuje
dokumentację dla czytelnika, który właśnie sklonował projekt.

## Co zostało dodane

### `NetFetcher` + `CurlFetcher`
`include/chesserazade/net_fetcher.hpp` definiuje abstrakcyjny
`NetFetcher { fetch(url) -> expected<string, FetchError> }`.
Produkcyjna implementacja, `CurlFetcher`, uruchamia systemowy
binarny `curl` przez `popen`. Utrzymuje to rdzenną bibliotekę
wolną od libcurl / OpenSSL / jakiejkolwiek zależności innej
firmy — HTTPS, TLS i obsługa przekierowań żyją po stronie
`curl`.

Bezpieczeństwo:
- URL jest walidowany: musi zaczynać się od `http://` lub
  `https://` i nie zawierać białych znaków / metaznaków
  powłoki, zanim zostanie interpolowany do stringa polecenia
  `popen`.
- `curl` jest wywoływany z `-s -f -L --max-time 30` —
  silent, fail przy HTTP ≥ 400, follow redirects, sufit 30s.

Testy używają `FakeFetcher` z spreparowanymi odpowiedziami;
realna ścieżka curl jest ćwiczona tylko przez CLI smoke testy.

### CLI: `fetch`
`src/cli/cmd_fetch.cpp`. Dwa tryby:

- `--url <URL>` — nieinteraktywny, do skryptów. Pobiera i
  zapisuje do `$XDG_CACHE_HOME/chesserazade/<hash>.pgn` (lub
  `$HOME/.cache/chesserazade/` jako fallback, lub `--out
  <path>` gdy wołający poda). Cache hit pomija sieć.
- Bez flag — interaktywne menu z trzema wyborami: wpisz surowy
  URL, pobierz puzzle Lichess po ID (konstruuje
  `https://lichess.org/api/puzzle/<id>`), lub anuluj. Każdy
  request sieciowy drukuje URL i prosi o potwierdzenie przed
  wysłaniem — reguła HANDOFF §8 „żadnego ruchu w tle".

Zcachowany plik jest przeznaczony do podania do `analyze
--pgn` albo komendy `load` w REPL.

## Czego **nie ma** w 1.0

- **Predefiniowane źródła poza Lichess-by-ID.** Interaktywne
  menu oferuje dokładnie dwa wybory: surowy URL i puzzle
  Lichess. Przyszła rewizja mogłaby dodać nazwane presety dla
  archiwów graczy z pgnmentor.com, bundli numerów TWIC itp.
- **Polityka inwalidacji cache'a.** Wpisy żyją wiecznie, chyba
  że użytkownik usunie katalog cache. Dla przypadków puzzle i
  statycznych gier to jest OK.
- **Równoległe / wsadowe pobierania.** Jeden URL naraz.
- **Requesty uwierzytelnione.** Tylko HTTPS GET — brak
  cookies, OAuth, bez body POST. Tylko publiczne, anonimowe
  źródła.

## Znane odstępstwa

- `curl` jest wywoływany przez `popen` a nie `posix_spawn`
  lub `fork`+`exec`, bo polecenie i tak leci przez powłokę
  (dla cudzysłowów wokół URL). URL jest walidowany jako
  ścisła forma HTTPS bez metaznaków przed interpolacją;
  defense-in-depth, URL nigdy nie opuszcza procesu CLI.
- `popen` / `pclose` to POSIX, nie ISO C++23; plik
  `curl_fetcher.cpp` włącza `_POSIX_C_SOURCE` lokalnie. To
  jedyne ustępstwo platformo-zależne w całej bazie.
