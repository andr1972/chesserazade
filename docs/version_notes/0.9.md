# 0.9 — Game analyzer

Tagged as **v0.9.0**.

0.8 made the engine competent at *finding* the right move from a
single position. 0.9 points that engine at a whole game: replay
the moves, judge each one, annotate the PGN.

## What was added

### `GameAnalyzer` (`include/chesserazade/game_analyzer.hpp`)
Walks a `Game` ply-by-ply. At each ply it searches both:
  * the pre-move position at the requested depth — this is the
    engine's verdict on what the mover *should* play;
  * the post-move position at depth-1, negated — this is what
    the played move actually achieves.

The centipawn *loss* (`best_score − played_score`) becomes a
PGN NAG suffix by a classical cut-off table:

```
  loss < 50 cp   → no annotation
  50 …  150      → "?!"   dubious
  150 … 300      → "?"    mistake
  > 300          → "??"   blunder
```

Mate swings override the cp table: "I had mate and didn't take
it" or "I walked into being mated" are always `??`.

For non-accurate moves the analyzer also produces a comment —
"best was Nc3 (+45cp)" — that the PGN writer puts in a `{…}`
block after the move.

### Annotated PGN writer
`write_pgn` gets a fourth overload that takes a
`std::vector<MoveAnnotation>` — one entry per ply, each with a
`suffix` (appended to the SAN token, e.g. `Nf6??`) and an
optional `comment` (emitted as a `{best was …}` block). The
original three-argument form is unchanged and continues to
compile for existing callers.

### CLI: `analyze`
```
chesserazade analyze --pgn <path> [--depth N]
```
Loads a single-game PGN, runs `GameAnalyzer` at the given depth
(default 10), emits an annotated PGN to stdout.

Example — Scholar's Mate at depth 4:
```
$ chesserazade analyze --pgn scholars.pgn --depth 4
[Event "Scholar"]
[Site "-"]
[Date "2025.01.01"]
[Round "-"]
[White "Alice"]
[Black "Bob"]
[Result "1-0"]

1. e4 e5 2. Bc4 Nc6 3. Qh5?! {best was Nc3 (+0cp)} Nf6??
 {best was Qe7 (+25cp)} 4. Qxf7# 1-0
```

Black's `Nf6??` is flagged as a blunder — not for a centipawn
swing, but because it walks into mate-in-1.

## Acceptance criteria (HANDOFF §9 0.9)

- [x] `analyze --pgn file.pgn --depth N` reports engine
      evaluation, best move, and blunder tags.
- [x] Annotated PGN output with NAG glyphs (`?`, `??`, `!?`).
      Positive NAGs (`!`, `!!`) are intentionally *not* emitted
      — see the "not in 0.9" section.
- [x] Produces a sensible annotated PGN for a known
      blunder-ridden sample game (Scholar's Mate), verified
      both by a unit test
      (`tests/test_analyzer.cpp`) and a CLI smoke test.

## What is *not* in 0.9

- **Positive NAGs (`!`, `!!`).** A "good move" is hard to
  define by a threshold alone — a move that simply matches the
  engine at trivial depth isn't worth a glyph. A principled
  heuristic (only-move detection, eval-jump) is out of scope
  for 0.9.
- **`{best line: …}` PV comments on every move.** The analyzer
  computes the PV, but the CLI only emits the comment for
  moves that got a NAG. A future `--verbose` flag can turn on
  every-move commentary.
- **Multi-PV.** 0.9 sees one main line per position.
- **Interactive analyzer (`repl` integration).** `repl` does
  not yet drive `GameAnalyzer`; analysis lives behind the
  `analyze` subcommand and is one-shot.

## Known deviations

- The comment on a walked-into-mate move shows the best
  move's *centipawn* evaluation (e.g. "best was Qe7 (+25cp)"),
  not a "mate avoided" phrase. The PGN is still correctly
  tagged `??`; only the comment is terser than it could be.
  Fixing this is a 1.0 polish task.
