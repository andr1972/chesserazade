# 0.1 — Foundations

Tagged as **v0.1.0**.

## What was added

### Build and tooling
- `CMakeLists.txt` with a `chesserazade_core` static library
  (shared by the executable and the tests) and a `chesserazade`
  console executable.
- `CMakePresets.json` with three presets: `debug`, `release`,
  `release-with-asserts` (RelWithDebInfo + `-UNDEBUG`). Ninja
  generator. `compile_commands.json` exported.
- `chesserazade_warnings` interface target centralizing
  `-Wall -Wextra -Wpedantic -Werror` + shadow / conversion / format /
  null-deref warnings; every target links it.
- `.clang-format` (LLVM-based, 100-column, snake_case-friendly).
- `.gitignore` expanded for CMake, build, and editor noise.

### Core types (`include/chesserazade/types.hpp`)
- `Color` (`White`, `Black`) with `opposite()`.
- `PieceType` including `None`.
- `Piece` = { `PieceType`, `Color` } with `none()` sentinel,
  structural equality, and `piece_to_fen_char` /
  `piece_from_fen_char` as the single source of truth for FEN
  piece encoding.
- `Square` under **little-endian rank-file mapping** (A1=0,
  H1=7, A8=56, H8=63, sentinel `None=64`).
- `File` and `Rank` enums.
- Helpers: `make_square`, `file_of`, `rank_of`, `to_algebraic`,
  `square_from_algebraic`.

### Move (`include/chesserazade/move.hpp`, `src/io/move.cpp`)
- Minimal `Move` struct for 0.1: `{ from, to, promotion }`.
  Flags for castle / en-passant / double-push are deliberately
  deferred to 0.2, where the move generator needs them.
- `to_uci(const Move&)` emits long-algebraic UCI:
  `e2e4`, `e7e8q`, `e1g1` (castling = king-move notation).

### Board (`include/chesserazade/board.hpp`,
### `src/board/board8x8_mailbox.{hpp,cpp}`)
- Abstract `Board` interface with the read-only query surface:
  `piece_at`, `side_to_move`, `castling_rights`,
  `en_passant_square`, `halfmove_clock`, `fullmove_number`.
- `CastlingRights` struct (four bools mirroring the FEN letters
  KQkq) with `any()`.
- `Board8x8Mailbox` concrete implementation: flat
  `array<Piece, 64>` plus scalar state. Mutators
  (`clear`, `set_piece_at`, setters) drive FEN and will drive the
  move generator in 0.2. Equality is structural.

### FEN (`include/chesserazade/fen.hpp`, `src/io/fen.cpp`)
- `FenError` error type with human-readable message.
- `STARTING_POSITION_FEN` convenience constant.
- `serialize_fen(const Board&)` — polymorphic, walks the
  abstract interface. Canonical castling order `KQkq`.
- `Board8x8Mailbox::from_fen(string_view)` returning
  `std::expected<Board8x8Mailbox, FenError>`. Strict ASCII.
  Accepts any permutation of castling letters on input; validates
  EP square to rank 3 or 6; rejects duplicate castling letters;
  rejects halfmove < 0 and fullmove < 1.

### CLI (`src/main.cpp`, `src/cli/`)
- Hand-rolled argument dispatch, one `cmd_*.cpp` file per
  subcommand.
- `chesserazade version` — prints `chesserazade 0.1.0`.
- `chesserazade show [--fen <fen>] [--unicode]` — renders a
  position. **ASCII by default**; `--unicode` opts into figurine
  glyphs (`♔♕♖♗♘♙…`). FEN input is ASCII only, regardless of
  `--unicode`, in line with the project convention.
- `chesserazade --help` / `<subcommand> --help` — subcommand-local
  help text.

### Tests (`tests/`)
- Catch2 v3.7.1 pulled via `FetchContent`.
- `test_types.cpp` — 10 cases covering Color inversion, Piece
  equality, FEN char round-trip and rejection, LERF invariants,
  algebraic round-trip, malformed-algebraic rejection.
- `test_move.cpp` — 4 cases covering quiet moves, promotions,
  castling notation, structural equality.
- `test_board.cpp` — 5 cases covering the empty default,
  set/get round-trip, `clear()`, setters via the abstract
  `Board&`, and structural equality.
- `test_fen.cpp` — 7 cases covering the starting position,
  all six standard perft FENs, en-passant, no-castling, partial
  castling, non-canonical castling normalization, and 13
  malformed-input rejections with message-substring assertions.
- **26 / 26 tests pass.**

### Docs
- `docs/coding_style.md` + `docs/coding_style_pl.md` (bilingual).
- `docs/version_notes/0.1.md` + `docs/version_notes/0.1_pl.md`
  (this file + Polish twin).
- `HANDOFF.md` and `docs/HANDOFF_pl.md` remain unchanged.

## Acceptance criteria (HANDOFF §9 0.1)

- [x] `chesserazade show --fen "<standard initial position FEN>"`
      renders the board.
- [x] FEN round-trip tests pass
      (starting position, all six perft positions, ep / castling
      variants).
- [x] Project scaffolding in place: CMake, presets,
      `.clang-format`, Catch2 wired.

## What is *not* in 0.1

The HANDOFF defers the following to later versions; they are
called out here so a reader of the 0.1 tree does not look for them:

- **Move generation** (0.2) — no pseudo-legal or legal moves yet.
  `make_move` / `unmake_move` also arrive in 0.2.
- **Perft** (0.2).
- **Special moves** — castling, en-passant, promotions are
  representable in FEN already, but no code plays them.
- **PGN** (0.4), **evaluation** (0.5), **search** (0.5+),
  **Zobrist / TT** (0.7), **puzzle solver** (0.8), **analyzer**
  (0.9), **fetcher** (1.0), **bitboards** (1.1), **GUI** (1.2).

## Known deviations from HANDOFF

- HANDOFF §11.5 recommended Unicode glyphs as the default for
  board rendering. We chose **ASCII default, `--unicode` opt-in**
  per user direction: FEN is an ASCII-only data format and the
  display default follows that constraint.
- HANDOFF §5 placed `board8x8_mailbox.hpp` under `src/board/`;
  it stays there. The static factory `Board8x8Mailbox::from_fen`
  keeps the public `fen.hpp` free of the concrete mailbox type so
  1.1 can add `BoardBitboard` cleanly.
- "CI stub" is deferred — the user is running this solo and
  prefers no CI wiring yet.
