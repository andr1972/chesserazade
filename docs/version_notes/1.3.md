# 1.3 — Qt6 graphical analyzer

Tagged as **v1.3.0**.

A separate executable target, `chesserazade-analyzer`, links
against `chesserazade_core` as a library. This is the **analysis
companion** to the CLI — not a play client — so it deliberately
talks to the engine directly (not over UCI) and exposes the
search's internal tree to the user.

## What was added

### 1.3.1 — PGN header indexer (`pgn_index.hpp`)
- O(n) walk over multi-game PGN text; extracts the Seven-Tag
  Roster + byte offset/length per game. No SAN parsing.
- Feeds the analyzer's game-list view in single-digit ms on a
  900-game Fischer archive.

### 1.3.2 — Search `TreeRecorder` + `BranchStats`
- Optional observer parameter on `Search::find_best`. Per-node
  `enter / leave` calls up to a configurable `ply_cap`.
- `BranchStats { captures_{w,b}, checks_{w,b} }` threaded
  through negamax. Captures tallied for free from
  `Move::captured_piece`; check counts gated on a recorder
  being attached (needs an after-move `is_in_check` probe).
- `SearchResult::pv_stats` — the PV's capture/check totals,
  available even without a recorder.

### 1.3.3 — Qt6 skeleton
- `find_package(Qt6 COMPONENTS Widgets Concurrent QUIET)`.
  Analyzer builds when Qt6 is present, silently skips
  otherwise.
- MainWindow with File → Fetch / Quit, Help → About.

### 1.3.4 — Fetch dialog (PGN Mentor)
- `data/pgnmentor_index.json` — curated 29-player snapshot.
- QDialog with QListWidget; cached entries render with a
  folder icon, remote entries with a file icon.
- Download via the existing `CurlFetcher`; unzip via the
  system `unzip` binary (hint-on-missing per distro).
- Cache layout mirrors the CLI's `cmd_fetch` so both tools
  share `~/.cache/chesserazade/`.

### 1.3.5 — Game-list view
- `GameListView` = sortable `QTableView` over the PGN index.
- Default sort: Date ascending.
- Double-click on a row emits `game_chosen` with that one
  game's bytes sliced out by offset/length.

### 1.3.6 — Board view + move list
- `BoardWidget` — custom-painted 8×8 grid, Unicode figurines
  (♙♘♗♖♕♔ / ♟♞♝♜♛♚), font scales with shortest side.
- `GameView` = QSplitter with `BoardWidget` + move-list
  (`QListWidget`, one row per ply with SAN via
  `chesserazade::to_san`).
- ←/→ step through plies, Home/End jump, Esc returns to the
  game-list.

### 1.3.7 — Solve panel
- Third page in MainWindow's `QStackedWidget`. Accessed via
  the "Solve from here" button on `GameView`, so search
  starts from whatever position the user was browsing.
- Budget picker: **Max ply / Time (s) / Nodes (× 1 000 or
  × 1 000 000)**.
- `SolveWorker` on a `QThread` — iterative deepening driven
  externally, emits a `progress` signal per completed depth
  with score, PV, `BranchStats`. UI stays responsive.
- Monospaced info log streams each `info` line plus a final
  `— done: bestmove … score … depth …` summary.

### 1.3.8 — Live search-tree view
- `SearchTree` arena (flat `std::vector<TreeNode>`), ~20 B per
  node + container overhead. Only plies `≤ ply_cap` are
  materialised — deeper search runs normally and bubbles up
  stats.
- `SearchTreeRecorder` is the concrete `TreeRecorder` that
  feeds the arena.
- `SearchTreeModel : QAbstractItemModel` exposes the tree to
  `QTreeView` with six columns: **Move (SAN) · Score · Capt
  W · Capt B · Chk W · Chk B**. Cutoff rows render in red with
  a `[cut]` suffix on the move name.
- The `SolvePanel` now houses both the info log and the tree
  in a vertical splitter. A "Tree ply cap" spinner lets the
  user trade tree detail against memory (default 3).
- Clicking a row in the tree replays its chain of moves onto
  the position held by `SolvePanel` and repaints the small
  `BoardWidget` — so "descend the tree" literally moves the
  figures on the analyzer's board.

## Scope decisions (per 1.3.0)

- **In-process, not UCI.** UCI does not expose the search
  tree; the whole point of this tool is that it does, so the
  analyzer links the engine as a library.
- **Single-threaded engine + no `stop` button.** The search
  respects its own budget and returns. Interruption is
  deferred.
- **Visible-tree nodes only.** `ply_cap` bounds memory. Below
  the cap the search runs normally and its captures / checks
  arrive as aggregates on the ply-cap boundary.
- **Cutoff icon = node that caused the cutoff** (fail-high,
  β-cut). Never-searched siblings simply do not appear.
- **Static PGN Mentor index.** Refresh by editing the JSON
  file; no live scraping.
- **Unzip via system binary.** Missing `unzip` yields a
  friendly install hint per distro.

## Known deviations

- **TT cuts reduce visible tree breadth.** At plies ≤ cap,
  early moves can return from the TT without exploring their
  subtrees, so those subtrees do not appear in the tree view.
  This mirrors what the engine actually did; a separate
  "fresh-TT pass" would show the full shape but double the
  work. Not on the 1.3 critical path.
- **Tree built on every ID iteration and discarded** — the
  final completed iteration's tree is what the view shows.
  Mid-iteration aborts reset the tree to the sentinel.
