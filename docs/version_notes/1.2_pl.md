# 1.2 — UCI (Universal Chess Interface)

Otagowane jako **v1.2.0**.

Zastępuje pierwotny plan 1.2 (GUI w Qt6). Zamiast pisać własny
GUI, `chesserazade` obsługuje teraz standardowy protokół UCI i
podłącza się do istniejących interfejsów szachowych (Arena,
CuteChess, SCID, …).

Referencja: https://wbec-ridderkerk.nl/html/UCIProtocol.html

## Co zostało dodane

### Podkomenda `chesserazade uci`
- Pętla liniowa stdin/stdout. Blokuje się na odczycie stdin,
  przetwarza jedną komendę na linię, wypisuje odpowiedzi na
  stdout z flushem po każdej linii (`std::ios::unitbuf`),
  kończy się na `quit` lub EOF.
- Implementacja w `src/cli/cmd_uci.{hpp,cpp}` jako publiczne
  `process_uci_line(UciSession&, line, out)`, dzięki czemu
  testy jednostkowe mogą sterować automatem stanowym bez
  odpalania podprocesu.

### Komendy GUI → silnik
- `uci` — odpowiada `id name chesserazade 1.2.0`,
  `id author Andrzej Borucki`, dwie linie `option` (`Hash`,
  `Threads`), `uciok`.
- `isready` — `readyok`.
- `ucinewgame` — czyści TT i resetuje szachownicę do startpos.
- `setoption name <N> value <V>` — akceptuje `Hash` (MiB,
  ograniczone do 1..4096) i `Threads` (no-op,
  jednowątkowo).
- `position {startpos | fen <fen>} [moves m1 m2 ...]` — parser
  FEN współdzielony z istniejącym `Board8x8Mailbox::from_fen`;
  ruchy w zapisie UCI long-algebraic (`e2e4`, `e7e8q`, …).
- `go depth N`, `go movetime MS`, `go infinite` oraz pełny
  kwartet kontroli czasu `wtime/btime/winc/binc` +
  opcjonalnie `movestogo`.
- `quit` — czyste wyjście. `stop` jest akceptowane i cicho
  ignorowane (patrz *Decyzje zakresowe*).

### Wyjście silnik → GUI
- Jedna linia `info depth N score cp X|mate N nodes N nps N
  time MS pv …` po każdej zakończonej iteracji iterative
  deepening.
- `bestmove <uci>` gdy search się kończy. Ruch pusty `0000`
  jest wypisywany, gdy pozycja startowa jest terminalna
  (mat lub pat).

### Zarządzanie czasem
- `go movetime MS` używa tego budżetu bezpośrednio.
- `go wtime/btime/winc/binc [movestogo]` mapuje się na
  movetime przez klasyczny wzór `budżet = czas / dzielnik +
  inc / 2`, przy czym `dzielnik = movestogo + 2` gdy GUI poda
  `movestogo`, w przeciwnym razie `30`. Dodatkowo cap na
  połowę pozostałego budżetu (minus 50 ms marginesu
  bezpieczeństwa) — silnik nigdy nie spali więcej niż połowy
  budżetu na jeden ruch.
- `go infinite` używa twardego capa `MAX_DEPTH` (32 ply).

### Iterative deepening z wyjściem info
- Handler `go` steruje ID z warstwy CLI: woła
  `Search::find_best` raz per głębokość docelową
  `d = 1..max`, współdzieląc `TranspositionTable` pomiędzy
  wołaniami i wypisując linię `info` po każdej zakończonej
  iteracji. `find_best` wewnętrznie iteruje od nowa `1..d`
  przy każdym wołaniu, ale TT sprawia że płytkie iteracje są
  prawie darmowe, a rdzeń silnika pozostaje nietknięty — bez
  callbacka progresu w `Search`.

## Decyzje zakresowe

- **Jednowątkowo.** Search blokuje czytanie stdin, więc `stop`
  nie ma robotnika do przerwania. Jest akceptowane i cicho
  ignorowane. Wątki + prawdziwe wsparcie `stop` są odłożone —
  silnik nadal respektuje swój własny budżet czasu / węzłów i
  wypisuje `bestmove` gdy budżet się kończy.
- **Wyjście info per zakończona głębokość**, nie per węzeł.
  Info per węzeł oznaczałoby callback z `Search`, czego
  aktualna architektura świadomie unika.
- **Brak multi-PV, pondera, Syzygy, UCI_ShowWDL, searchmoves.**
- **`stop` i nieznane komendy** są cicho ignorowane zgodnie z
  konwencją UCI (GUI rutynowo próbkują rozszerzenia).

## Ścieżka smoke-testu

```
printf 'uci\nisready\nposition startpos moves e2e4\ngo depth 6\nquit\n' \
    | chesserazade uci
```

Następnie wtyczka do Arena / CuteChess dla weryfikacji strony
GUI.

## Czego **nie ma** w 1.2

- **`stop`** — wymaga wątkowania (search na workerze, reader
  na main).
- **Ponder.**
- **Multi-PV.**
- **Tablice Syzygy, książka Polyglot.**
- **Prawdziwy GUI Qt6** (pierwotny plan 1.2, teraz odłożony
  bezterminowo — UCI sprawia że jest niepotrzebny do gry z
  zewnętrznymi GUI).
