# 0.8 — Sortowanie ruchów + quiescence + PuzzleSolver

Otagowane jako **v0.8.0**.

Trzy podręcznikowe dodatki zamieniają „poprawne, ale ślepe"
alfa-beta z 0.7 w rozpoznawalnie kompetentny search. Taktyki
pomijane na płytkich głębokościach zostają znalezione; hit rate
TT wyskakuje z ~2% do 30–50%; dedykowany punkt wejścia puzzle-owy
kończy „znajdowanie matów" w linii 0.x według HANDOFF.

## Co zostało dodane

### Sortowanie ruchów
`src/search/search.cpp` — nowe helpery `score_move` /
`score_and_sort` i `KillerTable` per ply.

Priorytety, od najwyższego:
```
  10'000'000   ruch z TT (probe w tym węźle)
   1'000'000 + MVV-LVA  — bicia (10 × ofiara − atakujący)
     900'000   pierwszy killer na tym ply
     800'000   drugi killer na tym ply
      50'000 + wartość promocji — ciche promocje
           0   zwykłe ciche ruchy
```

Killery są aktualizowane na każdym odcięciu beta, którego
wywołujący ruch **nie jest** biciem (bicia już i tak sortują się
wysoko przez MVV-LVA — wydanie rzadkiego slota na nie byłoby
redundantne). `KillerTable` jest alokowana raz per search i
przetrwa pomiędzy iteracjami iteracyjnego pogłębiania.

Referencje:
- https://www.chessprogramming.org/Move_Ordering
- https://www.chessprogramming.org/MVV-LVA
- https://www.chessprogramming.org/Killer_Heuristic

### Quiescence search
Gdy główny negamax osiąga `depth == 0`, wywołuje teraz
`quiesce(alpha, beta)` zamiast `evaluate(board)`:

```
quiesce(board, α, β):
    stand_pat = evaluate(board)
    if stand_pat ≥ β: return β
    if stand_pat > α: α = stand_pat
    for bicie in legalne bicia, posortowane MVV-LVA:
        make; s = -quiesce(-β, -α); unmake
        if s ≥ β: return β
        if s > α: α = s
    return α
```

Eksplorowane są tylko bicia (wraz z en-passant i promocjami-
biciami). Ciche ruchy nigdy nie uciekają poza horyzont, więc
rekursja kończy się w O(materiał) krokach. To zamyka klasyczny
*efekt horyzontu*: search kończący się w środku sekwencji bić
nie zwraca już eval sprzed-bicia jako finalnego.

Referencja: https://www.chessprogramming.org/Quiescence_Search

### PuzzleSolver (`include/chesserazade/puzzle_solver.hpp`)
Cienka otoczka nad `Search::find_best`, która dobiera
odpowiednią głębokość dla puzzle-a „mat w N":

```cpp
SearchResult PuzzleSolver::solve_mate_in(Board& b, int moves,
                                         TranspositionTable* tt = nullptr);
```

Wybór głębokości to `2N + 1` plyów:
- `2N − 1` to prawdziwa głębokość drzewa linii mat-w-N (N-ty
  ruch białych daje mata; zmatowana strona nie ma legalnej
  odpowiedzi na plyu `2N − 1`).
- Nasz negamax krótkim zwarciem przechodzi do `quiesce` przy
  `depth == 0` bez sprawdzenia terminalności, więc dokładamy
  jeden ply nadmiaru.
- Iteracyjne pogłębianie zatrzymuje się wcześnie na macie, więc
  dodatkowe ply są darmowe przy znalezionych matach.

### CLI: `solve --mate-in N`
`solve` zyskuje tryb puzzle:

```
$ chesserazade solve --fen "k7/8/8/1Q6/3K4/8/8/8 w - - 0 1" --mate-in 3
best:  Kd5  (d4d5)
score: mate in 3
depth: 6
pv:    1. Kd5 Ka7 2. Kc6 Ka8 3. Qb7#
nodes: 41433
tt:    28105 probes, 10546 hits (37.5%)
time:  119 ms
```

Przy nieudanym puzzle (brak matu w N w ramach głębokości),
`solve` drukuje `no mate in N found (best score …)` i kończy z
kodem 1 — przydatne dla skryptów i przyszłej integracji z REPL.

## Czego **nie ma** w 0.8

- **Heurystyka historii.** Licznik per (bierka, pole-cel)
  inkrementowany na odcięciu beta, używany do sortowania
  pozostałych cichych ruchów — klasyczna trzecia warstwa
  sortowania ruchów. Pominięte tu, bo killery + MVV-LVA + TT
  dają już redukcję węzłów której wymagał HANDOFF; dodanie
  historii to tweak na jeden plik w 0.9.
- **Rozszerzenia szachów / null-move pruning / LMR.** Poza
  zakresem linii 0.x zgodnie z mandatem dydaktycznym.
- **Szachy w quiescence.** Przeszukujemy tylko bicia. „Hałaśliwe
  nie-bicia" (szachy) dodałyby głębokości taktycznej, ale też
  eksplodowały drzewo; standardowa praktyka to ograniczenie za
  fail-soft testem wymijania szacha, co jest sprawą 1.x.
- **LMR oparte na heurystyce historii**, **rozszerzenia
  singularne**, **multi-PV** — nic z tego.
