# 1.0 — Internet fetching + documentation pass

Tagged as **v1.0.0**.

The "hand it to a stranger" milestone. The 0.9 engine was
already useful; 1.0 adds the last external-world affordance
(pulling games / puzzles from the internet) and puts the
documentation in shape for a reader who just ran `git clone`.

## What was added

### `NetFetcher` + `CurlFetcher`
`include/chesserazade/net_fetcher.hpp` defines an abstract
`NetFetcher { fetch(url) -> expected<string, FetchError> }`.
The production implementation, `CurlFetcher`, shells out to the
system `curl` binary via `popen`. This keeps the core library
free of libcurl / OpenSSL / any third-party dep — HTTPS, TLS,
and redirect handling live in `curl`'s lane.

Safety:
- The URL is validated to start with `http://` or `https://`
  and to contain no whitespace / shell metacharacters before
  being interpolated into the `popen` command string.
- `curl` is invoked with `-s -f -L --max-time 30` — silent,
  fail on HTTP ≥ 400, follow redirects, 30-second ceiling.

Tests use a `FakeFetcher` with canned responses; the real curl
path is exercised only by CLI smoke tests.

### CLI: `fetch`
`src/cli/cmd_fetch.cpp`. Two modes:

- `--url <URL>` — non-interactive, scripted. Downloads and
  writes to `$XDG_CACHE_HOME/chesserazade/<hash>.pgn` (or
  `$HOME/.cache/chesserazade/` fallback, or `--out <path>` if
  the caller specifies one). Cache hits skip the network.
- No flags — interactive menu with three choices: enter a raw
  URL, pull a Lichess puzzle by ID (constructs
  `https://lichess.org/api/puzzle/<id>`), or cancel. Every
  network call prints the URL and asks for confirmation
  before dispatching — HANDOFF §8's "no background traffic"
  rule.

The cached file is intended to be fed into `analyze --pgn` or
the REPL's `load` command.

## What is *not* in 1.0

- **Preset source definitions beyond Lichess-by-ID.** The
  interactive menu offers exactly two picks: raw URL and
  Lichess puzzle. A future revision could add named presets
  for pgnmentor.com player archives, TWIC issue bundles, etc.
- **Cache invalidation policy.** Entries live forever unless
  the user deletes the cache directory. For the puzzle and
  static-game use cases this is fine.
- **Parallel / batch downloads.** One URL at a time.
- **Authenticated requests.** HTTPS GET only — no cookies, no
  OAuth, no POST bodies. Public, anonymous sources only.

## Known deviations

- `curl` is invoked via `popen` rather than `posix_spawn` or
  `fork`+`exec` because the command already runs through a
  shell (for quoting of the quoted URL). The URL is validated
  as a strict HTTPS form with no metacharacters before
  interpolation; defense-in-depth, the URL never leaves the
  CLI process.
- `popen` / `pclose` are POSIX, not ISO C++23; the
  `curl_fetcher.cpp` file enables `_POSIX_C_SOURCE` locally.
  This is the single platform-specific concession in the
  codebase.

## Post-1.0 roadmap (per HANDOFF §9)

- **1.1** — `BoardBitboard` alternative implementation of the
  `Board` interface. Mailbox stays in-tree as the reference;
  a runtime flag (or CMake option) picks the implementation.
- **1.2** — Qt6 GUI front-end as a separate CMake target,
  linking the same `chesserazade_core`.

Deferred explicitly (see HANDOFF):
- UCI protocol adapter (so the engine can plug into Arena,
  CuteChess, SCID).
- Polyglot opening book as an optional read-only source.
- Syzygy endgame tablebase probing.
- SMP search (Lazy SMP or similar).
- **NNUE is out of scope** — it would undermine the classical,
  educational identity of the project.
