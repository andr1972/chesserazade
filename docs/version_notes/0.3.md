# 0.3 — Hardening the generator

Tagged as **v0.3.0**.

The generator itself shipped in 0.2 and passed the six standard
perft positions to depth 5. 0.3 is pure hardening: a benchmark,
an architectural reference document, and a final sweep over the
public header doc-blocks. No feature changes in the core engine.

## What was added

### `tools/perft_bench.cpp`
A stand-alone benchmark binary that runs perft on the six
standard positions and reports nodes, wall-clock time, and
Mnodes/sec per position. Not wired into tests, by deliberate
policy (HANDOFF §7: *"Benchmarks are not tests"*). Position
defaults are depth 6 for the initial position and position 3,
depth 5 for Kiwipete and positions 4-6, so a full run finishes
in a couple of minutes on a mailbox engine.

Options:
- `--depth N` — cap all positions at depth N.
- `--only NAME` — run a single named position
  (`initial`, `kiwipete`, `pos3`, `pos4`, `pos5`, `pos6`).

Sample timing on the reference machine in `release-with-asserts`:
```
initial   depth=6  nodes=     119060324  time=  14.18s  Mnps=  8.40
```

Built by the `CHESSERAZADE_BUILD_TOOLS` CMake option (on by
default). Target: `perft_bench`.

### `docs/architecture.md` (+ `architecture_pl.md`)
The reader's map of the codebase: layered diagram, data flow for
`show` and `perft`, key abstractions (`Board`, `Move` + `MoveKind`,
`MoveGenerator`, `MoveList`), directory layout, seven design
invariants, and extension points for 0.4+ subsystems. Complements
`HANDOFF.md`, which is forward-looking (plan), with a snapshot of
what is actually in the tree today.

### Doc-block sweep
All five public headers in `include/chesserazade/` were reviewed:
`types.hpp`, `board.hpp`, `fen.hpp`, `move.hpp`,
`move_generator.hpp`. Each carries a module-level doc-block with
purpose, invariants, and where applicable a link to the relevant
Chess Programming Wiki page. No changes were needed — the
doc-blocks were written to this standard as the modules shipped.
The review is recorded here for auditability.

## Acceptance criteria (HANDOFF §9 0.3)

- [x] Micro-benchmarks in `tools/perft_bench.cpp`.
- [x] Documentation pass: every public header has a doc-block;
      `docs/architecture.md` drafted.
- [x] Perft on standard positions to depth 6 completes in
      reasonable time (see timing table below — single-digit
      seconds per position in `release-with-asserts`).

### Depth-6 wall-clock (reference machine, `release-with-asserts`)

| Position  | Nodes           | Time    | Mnps |
| --------- | --------------- | ------- | ---- |
| initial   | 119,060,324     | ~14 s   | ~8.4 |

(Positions 2, 4, 5, 6 were not re-run at depth 6 for this table;
they default to depth 5 in the benchmark because depth 6 there
pushes into the tens of billions of nodes. Any reader can verify
with `perft_bench --only <name> --depth 6`.)

## What is *not* in 0.3

- **CI.** User preference (solo development). A later opt-in is
  trivial — a GitHub Actions YAML that runs
  `ctest --preset release-with-asserts` over the 32-case suite
  would be 30 lines.
- **Timing output inside the `perft` CLI command itself.** The
  benchmark prints it; the CLI `perft` still prints only the
  count. If a user starts wanting timing from the CLI rather
  than from `perft_bench`, adding a `--time` flag is a
  one-commit job in 0.4.
- **Anything touching the move generator's behavior.** That is
  frozen pending a found bug or the 1.1 bitboard rewrite.

## Known deviations
