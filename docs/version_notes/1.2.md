# 1.2 — UCI (Universal Chess Interface)

Tagged as **v1.2.0**.

Replaces the original 1.2 plan (Qt6 GUI). Instead of shipping
our own GUI, `chesserazade` now speaks the standard UCI protocol
and plugs into existing chess GUIs (Arena, CuteChess, SCID, …).

Reference: https://wbec-ridderkerk.nl/html/UCIProtocol.html

## What was added

### `chesserazade uci` subcommand
- Line-based stdin/stdout loop. Blocks reading stdin, processes
  one command per line, writes replies to stdout flushed per
  line (`std::ios::unitbuf`), exits on `quit` or EOF.
- Implemented in `src/cli/cmd_uci.{hpp,cpp}` as a public
  `process_uci_line(UciSession&, line, out)` so the test suite
  can drive the state machine without spawning a subprocess.

### GUI → engine commands supported
- `uci` — replies with `id name chesserazade 1.2.0`,
  `id author Andrzej Borucki`, two `option` lines (`Hash`,
  `Threads`), `uciok`.
- `isready` — `readyok`.
- `ucinewgame` — clears the TT and resets the board to startpos.
- `setoption name <N> value <V>` — accepts `Hash` (MiB,
  clamped to 1..4096) and `Threads` (no-op, single-threaded).
- `position {startpos | fen <fen>} [moves m1 m2 ...]` — FEN
  parsing shared with the existing `Board8x8Mailbox::from_fen`;
  moves in UCI long-algebraic notation (`e2e4`, `e7e8q`, …).
- `go depth N`, `go movetime MS`, `go infinite`, and the full
  time-control quartet `wtime/btime/winc/binc` + optional
  `movestogo`.
- `quit` — exit cleanly. `stop` is accepted and silently
  ignored (see *Scope decisions*).

### Engine → GUI output
- One `info depth N score cp X|mate N nodes N nps N time MS pv …`
  line per completed iterative-deepening iteration.
- `bestmove <uci>` when the search ends. The null move `0000`
  is emitted if the root position is terminal (mate or
  stalemate).

### Time management
- `go movetime MS` uses that budget directly.
- `go wtime/btime/winc/binc [movestogo]` maps to a movetime via
  the classical formula `budget = time / divisor + inc / 2`,
  with `divisor = movestogo + 2` when the GUI supplies
  `movestogo`, otherwise `30`. A cap of half the remaining time
  (minus a 50 ms flag-safety margin) clamps the result so the
  engine can never burn more than half of its budget on one
  move.
- `go infinite` uses the hard `MAX_DEPTH` cap (32 plies).

### Iterative deepening with info output
- The `go` handler drives ID from the CLI layer: it calls
  `Search::find_best` once per target depth `d = 1..max`,
  sharing the `TranspositionTable` across calls, and prints an
  `info` line after each completed iteration. `find_best`
  internally re-iterates `1..d` on every call, but the TT makes
  shallow re-runs almost free, and this keeps the engine core
  untouched — no progress-callback plumbing into `Search`.

## Scope decisions

- **Single-threaded.** The search blocks reading stdin, so
  `stop` has no worker to interrupt. It is accepted and
  silently ignored. Threading + true `stop` support are
  deferred — the engine still respects its own time / node
  budget and sends `bestmove` when that budget runs out.
- **Info output per completed depth**, not per node. Per-node
  info would mean a callback from `Search`, which the current
  architecture deliberately avoids (see above).
- **No multi-PV, no pondering, no Syzygy, no UCI_ShowWDL,
  no searchmoves.**
- **`stop` and unknown commands** are silently ignored per UCI
  convention (GUIs routinely probe for extensions).

## Smoke-test path

```
printf 'uci\nisready\nposition startpos moves e2e4\ngo depth 6\nquit\n' \
    | chesserazade uci
```

Output (shape):
```
id name chesserazade 1.2.0
id author Andrzej Borucki
option name Hash type spin default 16 min 1 max 4096
option name Threads type spin default 1 min 1 max 1
uciok
readyok
info depth 1 score cp 10 nodes 42 ...
info depth 2 score cp -40 ...
...
bestmove b8c6
```

Then plug into Arena or CuteChess locally to confirm the UI
side.

## What is *not* in 1.2

- **`stop`** — requires threading (search on a worker, reader
  on main).
- **Pondering.**
- **Multi-PV.**
- **Syzygy tablebases, Polyglot opening book.**
- **An actual Qt6 GUI** (the original 1.2 plan, now deferred
  indefinitely — UCI makes it unnecessary for external play).
